SECURITY & PRIVACY

OUR COMMITMENT

Frontend Protection (Flutter + Dart)

Our Flutter code is compiled into native binaries and obfuscated, making reverse engineering more difficult. We never expose API keys directly in the code; instead, we use encrypted configs and Firebase Remote Config.

Secure Communication (Network/API)

All connections use HTTPS to secure communications with Firebase and Alchemy. Our API keys are restricted by domain/IP to prevent misuse.

Firebase Security

Authentication is required to access sensitive resources. Firestore Security Rules ensure only authorized users access their data, and sensitive operations run via Cloud Functions securely.

Scraper Abuse Prevention

Request limits and input validation are in place to prevent abuse or injection attacks. We actively monitor usage and trigger alerts on suspicious activity.

Best Practices & Additional Security

We use Firebase App Check, enforce role-based access control, and keep detailed logs. These practices protect our users and ensure the integrity of our scraper platform.
